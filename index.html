Integrating with Web APIs

This by far has been the most difficult part to add to my blog. It took me two days just to figure out how to connect to the API for Giphy. During the first day I had no idea what was and wasn't working. That's when I figured out the importance of using console.log to show what few parts of my code actually work. I was finally able to send the request by changing the async to false.

Once I got a JSON response, I used console.log to figure out how to get the embed URL from all the data that was sent back. I was happy that I was finally able to figure it out but my work on this project was not complete. I received a message on the console that said Synchronous request are a no no so I needed to figure out how to change it to an Asynchronous request. Unfortunately, when I changed async to true, the code stoped working completely. A couple days and a couple dozen articles later I was able to format the request to work while async is true.

Together with JavaScript and Giphy's APIs I am able to give to you this random Gif generator. All you need to do is click the button and enjoy all the funny Gifs that Giphy has to offer.

Testing JavaScript

Finally... It has been about two weeks since I first said I was learning JavaScript and I have now learned enought to add something to my blog. The Mimo lesson didn't really even explain how to add my JavaScript functions to HTML. I have all this knowlege about how to create variables and loops but I had no idea how to save a function into a file and connect it into my blog.

But enough ranting, let me explain what I was able to do. I figured out that there are three ways to add JavaScript to HTML:

  1. Create a seperate JavaScript file and have an HTML line of code that points to the JS file
  2. Add the JavaScript code directly into the HTML file within the script & /script tags
  3. Pay a web developer to do it for you

In case you were wondering, I didn't accidentally display the date twice. The first date uses the first method that I listed and the second date has the script directly in the HTML code. Both scripts I wrote should update to display the date that you are vistiting the blog so I will have to check back tomorrow to see if it works
